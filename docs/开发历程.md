# TaskFlow AI 开发历程

> 从零到生产就绪，历时 6 天（2026-05-18 ~ 2026-05-23）

---

## 项目概述

基于 ReAct 模式的自主任务执行系统。用户提交自然语言任务 → LLM 规划步骤 → Agent 迭代选择并调用工具 → 合成最终结果。

**技术栈**：Java 21, Spring Boot 3.2, PostgreSQL, Redis, DeepSeek V4, OkHttp, JUnit 5

## 阶段一：核心引擎搭建（5/18）

提交 ReAct Agent 引擎的全部核心类：

- **Agent 层**：`TaskAgent`、`TaskAgentImpl`（Plan → Act → Observe → Reflect 循环）、`ExecutionContext`、`Plan`、`ExecutionStep`、`TaskResult`
- **LLM 集成**：`LLMClient`（文本 Prompt 调用 DeepSeek/OpenAI 兼容 API）
- **工具系统**：`Tool` 接口、`ToolManager`（含模糊匹配）、6 个内置工具
  - `calculator` — 数学表达式计算
  - `weather` — 天气查询（WeatherAPI）
  - `search` — 网络搜索（Tavily）
  - `database` — SQL 查询
  - `file_operation` — 文件读写/列表/删除
  - `json` — JSON 解析/查询/转换

**关键设计决策**：不用原生 function calling，用纯文本 Prompt + JSON 返回 `selected_tool`。ToolManager 通过去下划线、小写化做模糊匹配，处理 LLM 返回的工具名变体。

## 阶段四：生产就绪（5/23）

### 数据库迁移（Flyway）

| 变更 | 说明 |
|------|------|
| 添加 `flyway-core` 依赖 | 版本化 SQL 迁移 |
| 创建 `V1__init.sql` | 基线 DDL（task_executions + execution_steps 两张表 + 3 个索引）|
| `ddl-auto: update` → `validate` | Hibernate 不再自动改 schema |
| `baseline-on-migrate: true` | 首次引入 Flyway 到已有数据库 |

### API 限流（Resilience4j）

| 变更 | 说明 |
|------|------|
| 添加 `resilience4j-spring-boot3` | RateLimiter + Spring AOP |
| `@RateLimiter(name = "agent-execute")` | 加在 executeTask / executeTaskAsync |
| 默认 10 req/s | 1 秒刷新周期，超限立即拒绝 |
| `RequestNotPermitted` → 429 | 响应体含 `errorCode: RATE_LIMITED` |

### 压力测试

| 场景 | 并发 | 请求数 | 结果 |
|------|------|--------|------|
| Mock 高并发 | 20 线程 × 5 任务 | 100 | 全部完成，耗时 4.7s |
| 幂等性（同 Key） | 10 | 10 | 1×202 + 9×409 |
| 幂等性（不同 Key） | 10 | 10 | 10×202 |
| API 限流 | 20 | 20 | 10×202 + 10×429 |

**发现并修复**：幂等性过滤器的 `isDuplicate()` → `store()` 不是原子操作，并发下同一 key 可被多个请求同时通过。改用 `ConcurrentHashMap.putIfAbsent()` + Redis `SET NX` 实现原子 check-and-set。

### ReAct 循环流程

1. **Plan** — LLM 分析任务，生成步骤计划
2. **Act** — 每步由 LLM 选择工具 + 参数，ToolManager 解析并执行
3. **Observe** — 工具结果写入 ExecutionContext，全量历史写入下一轮 Prompt
4. **Reflect** — LLM 评估结果是否满足任务，决定继续或结束
5. 循环直到完成 / 超时 / 安全阀触发

### 三层循环终止保护

- LLM 决策层：action 中的 `should_continue` + reflection 中的 `is_sufficient`
- 代码安全阀：同工具 + 同参数重复检测（SHA-256 hash）
- 系统限制：超时 + 最大步数上限
